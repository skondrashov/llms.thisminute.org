What is a harness?

A harness is the software layer between you and a model. It decides what goes into the context window, which tools are available, how the conversation is managed, and when to stop and show you something. ChatGPT, Cursor, Claude Desktop, GitHub Copilot, Microsoft 365 Copilot, Grammarly: those are all harnesses, even though they can look and feel very different from each other.

Without a harness, a model is a function that predicts the token most likely to follow a given context. The harness is what turns that into a usable system: it runs the conversation loop, exposes tools the model can request, executes those tool calls, decides how to present responses, and keeps the whole cycle going until the task is done or the context runs out.

When a model feels like an agent rather than a chatbot, that is usually the harness at work. The system prompt, the tool definitions, the decision to summarize old context or start a fresh session, the way errors get retried: all of that is harness code, not model behavior. Swap the same model into a different harness and the experience changes dramatically, even though the weights haven’t moved.

Differences between harnesses

Harnesses come in a lot of shapes, but they tend to differ on a few things that matter in practice:

  • Setting. Where does it live? A CLI you run in a terminal, an IDE extension wired into your editor, a web app in a browser tab, a desktop app, or an SDK you embed in your own code. The setting shapes everything: what tools the harness can reach, how it presents output, how fast the feedback loop feels.
  • Model support. Some harnesses are locked to one vendor (ChatGPT only runs OpenAI models, Claude Code only runs Claude). Others let you plug in whatever model you want. This matters because model choice directly affects quality, and being locked to one vendor means you can’t swap when a better option shows up.
  • Tool ecosystem. What can the harness actually do beyond chatting? File reads, terminal commands, web search, code execution, MCP servers. The tool set is often the real differentiator. A harness with strong tool support turns a model into an agent; one without is just a chat box.
  • Context management. How does the harness handle long conversations? Does it silently truncate old messages, summarize them, let you start fresh? This is invisible until it isn’t, and it’s one of the hardest problems in harness design. See context.
  • Pricing model. Some harnesses charge a flat subscription (Cursor, ChatGPT Plus). Others pass through API costs directly (Claude Code, Aider). Some are free and open source. The pricing model changes how you use the tool, because per-token costs make you think twice about long agent loops in a way that a monthly fee doesn’t.

Harness vs model vs agent

People confuse these three constantly, and it’s worth getting straight. A model is the engine: a pile of trained weights that predicts the next token. A harness is the car built around the engine: the steering wheel, the dashboard, the gas pedal, everything you actually touch when you drive. An agent is less a third thing and more a mode of operation: the harness running in a loop, calling tools, making decisions without waiting for you at every step, with the model generating each next move.

When someone says "I asked ChatGPT to do X," they’re talking about a harness. ChatGPT is a web-based harness built by OpenAI. The model inside it might be GPT-5 or GPT-5.5 or o3, depending on what they selected. The harness is what formatted their message, managed the conversation history, and decided how to present the reply. If they turned on "deep research" mode and the thing started browsing the web and writing a report, that’s the harness running an agent loop with the model inside it.

This distinction matters because you can put the same model in different harnesses and get wildly different results. Claude Sonnet 4.6 inside Claude Code (a CLI harness with full filesystem access) behaves nothing like Claude Sonnet 4.6 inside the claude.ai web interface (a chat harness with limited tools). Same engine, different car, different driving experience.
