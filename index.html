LLMs!

When you use an LLM, you are (almost always) interacting with it through something called a harness. A harness coordinates 3 distinct things: the LLM itself, which reasons and generates text, a context which tailors the LLM to your needs, and tools that allow the system to read, write, and otherwise interact with its environment.

harness

The outer loop. Manages the context window, calls tools, runs the agent cycle, and decides when to stop. The thing you actually talk to.

Large language model f(tokens) → token The model itself only has one operation: given a context, output a token. In a lot of ways, it is astonishing that this simple task has unlocked so many secrets of language and intelligence.
Context the window The single input to the model. Everything it knows about you, the task, and the conversation has to fit in here. There is no other memory. When it fills up, something has to go.
Tools files, APIs, the web A model alone cannot read files, browse the web, or run code. Tools are functions the harness exposes. The model requests them; the harness executes them and feeds the results back into the context.
