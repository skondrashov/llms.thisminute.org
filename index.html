LLMs!

People generally interact with LLMs through harnesses. A harness coordinates 3 distinct things: the LLM itself, which reasons and generates text, a context which tailors the LLM to your needs, and tools that allow the system to read, write, and otherwise interact with its environment.

harness

A coordination system that connects a user to an LLM and manages the context window, calls tools, runs the agent cycle, and more.

Large language model The model can only do one operation: given a context, output a token. It is completely static in normal operation, and does not update or learn anything. That is what the context is for!
Context A set of tokens that serve as input to the model. Everything a model needs to know about the current user, task, or conversation must be represented here one way or another.
Tools A model alone cannot read files, browse the web, or run code. Tools are functions the harness exposes. The model requests them; the harness executes them and feeds the results back into the context.
orchestration

Patterns for coordinating multiple context windows, agents, and workflows.
