What is a model?

A model is a pile of trained weights that does exactly one thing: given a sequence of tokens, it predicts the next one. There’s no reasoning engine in there, no agent, no “it” in the human sense. The whole experience of talking to an AI is built by running that single next-token prediction over and over, with the model’s own output feeding back into the input, until it produces a stop token.

When we say a model is GPT-5 or Claude Sonnet 4.6 or Llama 3.1, we’re talking about a specific set of weights. Swap the weights and the behavior changes completely. The harness (Claude Code, Cursor, Aider), the context window, the tools, all of that can stay the same, and you’ll get a different agent.

That’s why model choice matters so much. In agentic engineering the model is the engine, and the rest is the car built around it.

Differences between models

Every model has the same general shape, but they differ on a handful of things that matter for agent work:

  • Size and capability. Bigger weights usually mean stronger reasoning and better recall, at higher cost and slower latency. A vendor typically has a large / medium / small tier (Opus / Sonnet / Haiku, GPT-5 / Mini / Nano, and so on) that trades quality for speed.
  • Context window. How much the model can read at once, measured in tokens. Current range is roughly 128K to 2M. For agentic work, where tool output dominates, bigger is almost always better. See context.
  • Thinking modes. Some models produce reasoning tokens before the final answer (o3, Claude with extended thinking, Gemini 2.5 Pro). They tend to get smarter the more tokens they burn before committing to a reply.
  • Vision and multimodality. Most frontier models accept images as input, and a growing share take audio. Each modality gets turned into tokens and processed in the same window.
  • Hosted versus open weights. Hosted models live behind an API and you pay per token. Open-weights models (Llama, Mistral, DeepSeek, Qwen) ship the weights, so you can run them yourself, fine-tune them, or inspect them. The frontier is still on hosted, but the gap narrows every cycle.

Quirks worth knowing

None of this is in the marketing copy, but all of it affects how agents built on these models behave.

The context isn’t uniformly attended. Even inside a 1M window, tokens near the beginning and the end tend to have more influence than tokens in the middle. Good harnesses bury important context at the tail for exactly this reason.

Models from the same vendor share a tokenizer and a training lineage, so Claude 3.5 Sonnet and Claude 4.6 Sonnet feel more alike than Claude and GPT-5 do, even at similar benchmark scores. Different vendors have different personalities, different refusal patterns, different habits around tool-calling. Switching a big agent system from one vendor to another always turns up surprises.

The sticker-price cost of a model is almost never the real cost. An agent loop with a thinking model spends way more tokens on reasoning than on the user-visible reply, so a cheap-per-token model that needs three loops can end up more expensive than a premium model that gets it in one shot. Benchmarks measure correctness, not the price of the loop.

And the field moves fast. A model that was frontier a year ago is a cheap workhorse today, and today’s frontier will be a workhorse by next year. Production systems usually pin to a specific version, because the “same model name” can silently shift behavior after a vendor-side update.
